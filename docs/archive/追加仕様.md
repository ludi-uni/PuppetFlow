# Phase 10

## PuppetFlow Ecosystem

OSS完成形

---

パッケージ群

```text
@puppetflow/core
@puppetflow/runtime

@puppetflow/plugin-rule
@puppetflow/plugin-emotion
@puppetflow/plugin-idle

@puppetflow/source-mqtt
@puppetflow/source-http
@puppetflow/source-discord

@puppetflow/adapter-vmc
@puppetflow/adapter-vrm

@puppetflow/studio
```

---

この時点でのPuppetFlowは、

```text
キャラクター制御ライブラリ
```

ではなく

```text
イベント駆動型キャラクターランタイム
```

になります。

そしてDOLLはその上に載る一つの実装例です。

実はこの構造、DOLLの「観測→状態→振る舞い→表現」という思想とも綺麗に一致しています。だからPhase 7以降はDOLL向け機能を追加しているつもりでも、結果的にPuppetFlow全体の価値が上がるはずです。むしろ危険なのは早い段階でDOLL専用機能をコアに混ぜることです。そこだけは避けた方が長生きする設計になります。
