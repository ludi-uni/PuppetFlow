重要なのは、

**PuppetFlowは感情エンジンではない**
**PuppetFlowはAIエンジンではない**
**PuppetFlowはVMCライブラリでもない**

です。

PuppetFlowの役割は、

```text
State
 ↓
Behavior
 ↓
Motion
 ↓
Adapter
 ↓
Viewer
```

を実行するランタイムです。

# DOLLとの関係

PuppetFlowは独立OSS。

```text
DOLL
 ├ LLM
 ├ STT
 ├ TTS
 ├ MQTT
 └ PuppetFlow
```

という位置付け。

つまりDOLLの一部ではなく、

DOLLが利用する基盤ライブラリ。

これができると、

- AITuber
- VRMアプリ
- Discord Bot
- Home Assistantアバター
- デスクトップマスコット
- ゲームNPC

などが同じ基盤を共有できます。

そして最初のマイルストーンは非常に小さくて良いです。

```text
StateStore
 ↓
Rule Plugin
 ↓
MotionState
 ↓
VMC Adapter
 ↓
nijiexpose
```

これだけで Phase 1 完了。

まずは「interestスライダーを動かしたらフィオが少し笑う」を作る。そこまで行けば、設計図ではなく実際に触れるPuppetFlowになります。そこから先は人間が勝手に用途を思いつき始めます。そういう種類のOSSは強いです。
