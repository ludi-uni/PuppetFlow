# PFScript仕様

## PuppetFlow Behavior Language

---

PFScriptは

- Scratch Editor
- Graph Editor

```text
Lua風DSL
```

# サンプル

```lua
-- 興味に応じて笑顔

smile =
    interest * 0.4

-- 音量で口パク

mouthOpen =
    volume

-- 軽いゆらぎ

headTilt =
    noise(time * 0.2) * 0.1

-- 考え込み状態

if interest > 0.7 then

    thinking(
        intensity = 0.8
    )

end

-- 音素リップシンク

if currentPhoneme == "A" then
    MouthA = 1
end

if currentPhoneme == "I" then
    MouthI = 1
end
```

この仕様なら、

- 初心者 → Scratch
- 中級者 → Graph
- 上級者 → PFScript

を同じランタイム上で共存させられます。

そしてPFScriptは「Lua互換言語」ではなく、「キャラクターの振る舞いを書くための小さなDSL」として保つのが重要です。ここを欲張ると、そのうちPuppetFlowの半分が言語処理系になってしまいます。人類はなぜか自作言語を育て始めると止まらないので、最初から柵を立てておくのがおすすめです。

## 実装メモ（Phase 1）

> **2026-06 時点** — 実装詳細は [reference/pfscript.md](reference/pfscript.md) / [implementation-plan-pfscript.md](implementation-plan-pfscript.md) を正とする。

### 実装済み

- Lexer / Parser / Lowering / Preset `behaviorPfScript` / Studio PFScript タブ
- 組み込み関数（Math / Trig / `noise` / `lerp` / `eventActive`）
- Motion エイリアス（`smile`→`mouthX`, `mouthOpen`→`mouthY` 等）
- 条件付き `thinking()` — Behavior 実行経路で If 評価後に Extension Pack へ

### 仕様との差分（Phase 1 未対応）

| 仕様上の例 | Phase 1 の扱い |
| ---------- | -------------- |
| `lerp(smile, targetSmile, 0.1)` | **非対応** — ローカル変数なし |
| `thinking(intensity = interest * 0.5)` | 引数は **数値リテラルのみ** lower |
| `currentPhoneme` | Channel キーは **`phoneme`** も参照 |
| `browUp` / `breath` 等の Motion 名 | エイリアス未定義 → **`custom`** へ |
| PFScript ↔ Graph 自動変換 | **非対応** |

### Motion エイリアス表（実装）

| PFScript | MotionState |
| -------- | ----------- |
| `smile` | `mouthX` |
| `mouthOpen` | `mouthY` |
| `eyeOpen` | `eyeYaw` |
| `eyeSmile` | `eyePitch` |
| 標準 14 キー同名 | そのまま |
| 上記以外（`MouthA` 等） | `custom.*` |

### Pack 二重実行について

PFScript の `thinking()` と Preset `extensions.packs` の **同一 Pack ID** は Studio で警告。実行は両経路とも Extension Layer に到達しうるため、重複を避けてください。
