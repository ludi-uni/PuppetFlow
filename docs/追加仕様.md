これはかなり重要な拡張です。

今までのPFScriptは

```text
State
↓
計算
↓
Motion
```

という**関数型モデル**でした。

しかし実際のキャラクター表現は、

```text
前フレーム
↓
現在フレーム
↓
次フレーム
```

を利用するものが非常に多いです。

つまり、

```text
Behavior Runtime
```

だけでなく

```text
Behavior Runtime + Internal State
```

が必要になります。

# 設計方針

PuppetFlowは

```text
アニメーション再生ツール
```

ではなく

```text
状態から
自然な振る舞いを生成する
モーションランタイム
```

である。

そのため、

単純な数式だけでなく、

```text
時間
履歴
慣性
ランダム性
```

を扱う Stateful Node をコア機能として提供する。

---

特に **Smooth・Oscillator・Spring** の3つは最重要です。

実際、Live2D・Inochi2D・VRM・ゲームAIの待機モーションの大半は、この3つの組み合わせでかなり表現できます。人類は複雑な物理演算を欲しがりますが、見た目の8割は「補間」「振動」「慣性」でできています。そこを先に揃えるとPuppetFlowの価値が一気に上がります。
