# PuppetFlow 追加仕様

## Input Channel & Timeline System

---

# 目的

PuppetFlowは

```text
State
↓
Behavior
↓
Motion
↓
VMC
```

を担う。

---

一方、

- LLM
- TTS
- MQTT
- Home Assistant
- ゲームイベント

などは外部システムとする。

---

そのため、

PuppetFlowは

```text
入力ソース
```

ではなく

```text
入力データ
```

を扱う。

---

# 設計方針

PuppetFlowは

```text
STT Framework
```

ではない

---

PuppetFlowは

```text
TTS Framework
```

でもない

---

PuppetFlowは

```text
VMCベースの
キャラクターモーションランタイム
```

である

---

そのため、

```text
Whisper
Gemini
GPT
VOICEVOX
AivisSpeech
OpenUtau
MQTT
```

は外部システムとして扱う。

---

PuppetFlowが責任を持つ範囲は

```text
State
Channel
Timeline

↓

Behavior

↓

MotionState

↓

VMC
```

までとする。

これで「AIそのもの」ではなく「キャラクターがどう動くか」に集中できます。DOLLのような上位システムから見ると、PuppetFlowは感情・音声・イベントを受け取り、Live2DやInochi2Dへ自然なモーションとして流し込む共通ランタイムになります。そこがプロジェクトの芯です。
