それは入れるべきです。

むしろ今の仕様だと、

```text
smile
eyeOpen
headTilt
```

などの標準パラメータしか触れません。

しかし実際には、

```text
考え込むモーション
待機モーション
視線探索
尻尾を振る
耳を動かす
羽を動かす
```

みたいな「動作パッケージ」が欲しくなります。

---

ただし重要なのは、

**プラグインがMotionStateを直接変更しないこと**

です。

ここを許すと後でカオスになります。

# 設計方針

プラグインは

```text
State
Channel
Timeline
```

を書き換えない。

---

プラグインは

```text
Parameter
Function
Node
Motion Pack
```

を拡張する。

---

Behavior Runtimeは常に共通。

---

つまり、

```text
PuppetFlow Core
```

は固定し、

```text
キャラ固有の個性
特殊部位
待機動作
演出
```

はプラグインとして追加する。

これを入れると、フィオ向けの「考え込む」「観測中」「興味を持つ」もプラグインになりますし、将来誰かが「ケモミミ用耳ぴくぴくパック」や「猫しっぽパック」を作ってもコアは一切変更不要です。

そして重要なのは、**PFScript・Scratch・Graphのどの編集方式からも同じプラグイン機能が見えること**です。そうするとエディタだけ違って中身は共通という、かなり綺麗なエコシステムになります。
